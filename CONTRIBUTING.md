# Contributing to Lag Clear

Thank you for your interest in contributing to Lag Clear! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an [issue](../../issues/new) with:
- A clear description of the problem
- Steps to reproduce it
- Your Minecraft and mod version
- Any relevant log files

### Suggesting Enhancements

We love new ideas! Please feel free to:
- Open an issue with your suggestion
- Explain why this enhancement would be useful
- Provide examples if possible

### Submitting Code

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** and test them thoroughly
4. **Commit with clear messages**: `git commit -m "Add: description of changes"`
5. **Push to your fork**: `git push origin feature/your-feature-name`
6. **Open a Pull Request** with a clear description of your changes

### Code Standards

- Follow existing code style and conventions
- Write clear, descriptive commit messages
- Test your code before submitting
- Keep pull requests focused on a single feature/fix

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/LagClear.git
cd LagClear

# Build the project
./gradlew build

# Run in development
./gradlew runClient
```

## Questions?

Feel free to open an issue if you have any questions about the contribution process!
